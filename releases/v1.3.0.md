## TL;DR

为组件库引入完整 a11y 能力层：axe-core 烟雾测试 + jest-dom v6 契约断言，并入 CI 门禁；并修复 3 个组件级 a11y 缺口（Form / Input / Progress）。**无破坏性变更**。

---

## ✨ Added

### a11y 基础设施

- **`test:a11y` 脚本**：基于 `vitest-axe` 对 28 个组件跑 axe-core 烟雾测试，输出规则违规列表
- **CI 门禁**：`npm run ci` 串新增 `test:a11y` 节点（format:check → check:docs → lint → test:run → test:a11y → build）
- **`vitest.a11y.config.ts`**：独立 vitest 配置，仅包含 a11y 套件，隔离主 CI
- **新 devDeps**：`axe-core@^4.12.1`、`vitest-axe@^0.1.0`

### a11y 契约层（jest-dom v6）

在 13 个组件测试里就近补 `toHaveAccessibleName` / `toHaveRole` / `toBeInvalid` / `toHaveAccessibleErrorMessage` / `toHaveAccessibleDescription` 断言 —— 与 axe 烟雾形成**广度（WCAG 规则）+ 深度（组件契约）**双层防御。

覆盖组件：Button / Icon / Switch / Radio / Checkbox / Tabs / Collapse / Input / Select / Form / Modal / Drawer / Notification / Tooltip / Tag / Card / Progress / Table

### Progress

- 新增 `aria-label` / `aria-labelledby` props，透传到 `role="progressbar"` 根节点
- 修复 axe 规则 `aria-progressbar-name` 违规

## 🐛 Fixed

### Form / FormItem

- **id 自动绑定**：FormItem 克隆子元素时自动注入 `id={fieldKey}`，与 `<label htmlFor={fieldKey}>` 配对
    - 用户已显式传 `id` 时不覆盖（与 `disabled` / `size` / `status` 透传策略一致）
- **错误态可访问性**：给 clone 出的 child 注入 `aria-errormessage={helpId}`，帮助节点加 `id={helpId}`，让屏幕阅读器能朗读错误信息

### Input

- `status="error"` 时输出 `aria-invalid="true"` 到 `<input>` 元素

## ⬆️ Upgrade Notes

**无破坏性变更**，所有改动为加法。典型用户代码无需任何调整。

### 兼容性细节

- **Form/FormItem 的 `id` 注入**：仅在 `name` 设置时生效，且当 child 已有 `id` 时不覆盖。极边角情况：若你的自定义子组件内部对 `id` 有自管理逻辑（如 `useId` 或自生成 id），需要确认与 `name` 的命名空间不冲突 —— 这种情况在 28 个内置组件中均不存在。
- **Progress 新 props**：纯加法，调用方式不变。
- **Input 的 `aria-invalid`**：仅在 `status="error"` 时输出，对其他状态完全无影响。
